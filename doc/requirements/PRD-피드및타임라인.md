# PRD: 피드 및 타임라인

## 기능 개요
사용자가 팔로우한 사람들의 게시물을 시간순 또는 추천순으로 볼 수 있는 메인 피드 기능입니다. 서비스의 핵심이 되는 화면입니다.

## 우선순위: 🔴 필수

## 상세 기능 명세

### F5-1. 메인 피드
| 항목 | 내용 |
|------|------|
| 기능명 | 메인 피드 (타임라인) |
| 설명 | 사용자가 팔로우한 사용자들의 게시물과 리포스트가 타임라인에 표시된다. |
| 입력 | 앱 접속 시 자동 로드 |
| 처리 | Firestore에서 팔로우 관계 기반 게시물 쿼리, 실시간 업데이트 수신 |
| 출력 | 게시물 카드 리스트 (작성자 정보, 본문, 미디어, 반응 수) |
| 예외 | 팔로우한 사용자가 없는 경우 → 추천 게시물/사용자 표시 |

### F5-2. 피드 정렬
| 항목 | 내용 |
|------|------|
| 기능명 | 피드 정렬 방식 전환 |
| 설명 | 사용자는 피드를 '최신순' 또는 '추천순'으로 전환하여 볼 수 있다. |
| 입력 | 정렬 탭 전환 (최신순 / 추천순) |
| 처리 | 최신순: 작성 시각 기준 내림차순, 추천순: 좋아요·댓글·리포스트 수 기반 가중치 정렬 |
| 출력 | 선택된 기준에 따라 재정렬된 피드 |
| 예외 | 추천순 데이터 부족 시 → 최신순으로 대체 |

### F5-3. 무한 스크롤
| 항목 | 내용 |
|------|------|
| 기능명 | 무한 스크롤 (페이지네이션) |
| 설명 | 사용자가 피드를 아래로 스크롤하면 이전 게시물이 자동으로 추가 로드된다. |
| 입력 | 스크롤 이벤트 |
| 처리 | Firestore 커서 기반 페이지네이션 (startAfter) |
| 출력 | 스크롤 시 추가 게시물 로딩 |
| 예외 | 더 이상 게시물이 없으면 '모두 확인했습니다' 메시지 |

## 화면 구성

| 화면 | 주요 요소 |
|------|-----------|
| 메인 피드 | 상단 정렬 탭 (최신순/추천순), 게시물 카드 리스트, 새 게시물 작성 FAB 버튼 |
| 게시물 카드 | 작성자 프로필(사진, 닉네임), 본문(마크다운), 미디어, 액션 바(댓글·좋아요·리포스트·북마크) |
