# PRD: 그룹 및 커뮤니티

## 기능 개요
사용자가 관심 분야나 지역을 기반으로 그룹을 생성·가입하여, 특정 주제에 대한 심도 있는 토론과 정보 공유를 할 수 있는 기능입니다.

## 우선순위: 🟡 중요

## 상세 기능 명세

### F6-1. 그룹 생성
| 항목 | 내용 |
|------|------|
| 기능명 | 그룹 생성 |
| 설명 | 사용자는 특정 기술 주제(예: LLM 연구, 프론트엔드 개발)나 지역을 기반으로 그룹을 생성할 수 있다. |
| 입력 | 그룹명, 설명, 카테고리(관심 분야/지역), 공개/비공개 설정, 그룹 이미지 |
| 처리 | Firestore에 그룹 문서 생성, 생성자를 관리자로 설정 |
| 출력 | 그룹 페이지 생성 완료 |
| 예외 | 그룹명 중복 → 안내 메시지 |

### F6-2. 그룹 가입/탈퇴
| 항목 | 내용 |
|------|------|
| 기능명 | 그룹 가입/탈퇴 |
| 설명 | 사용자는 공개 그룹에 자유롭게 가입하거나, 비공개 그룹에는 가입 요청을 보낼 수 있다. |
| 입력 | 가입/탈퇴 버튼 클릭 |
| 처리 | 공개: 즉시 멤버 추가, 비공개: 관리자 승인 후 추가 |
| 출력 | 가입된 그룹 목록에 표시, 그룹 피드 접근 가능 |
| 예외 | 이미 가입한 그룹 → 탈퇴 버튼 표시 |

### F6-3. 그룹 전용 포스팅 및 토론
| 항목 | 내용 |
|------|------|
| 기능명 | 그룹 내 포스팅 |
| 설명 | 그룹 멤버만 볼 수 있는 배타적인 게시물을 작성하고 토론할 수 있다. |
| 입력 | 그룹 피드에서 게시물 작성 |
| 처리 | 게시물의 공개 범위를 해당 그룹으로 제한하여 Firestore에 저장 |
| 출력 | 그룹 피드에서만 해당 게시물 표시 |
| 예외 | 비멤버가 접근 시 → '가입이 필요합니다' 안내 |

### F6-4. 지역 기반 그룹
| 항목 | 내용 |
|------|------|
| 기능명 | 지역 기반 그룹 |
| 설명 | 사용자가 직접 선택한 지역 정보를 바탕으로 '서울 AI 스터디', '부산 개발자 모임' 등의 지역 기반 그룹을 형성할 수 있다. |
| 입력 | 지역 선택 (드롭다운/검색) |
| 처리 | 사용자가 선택한 지역 정보를 기반으로 그룹 생성/검색 |
| 출력 | 지역 필터가 적용된 그룹 목록 |
| 예외 | 지역 정보 미설정 시 → 지역 그룹 추천 비활성화 |

## 화면 구성

| 화면 | 주요 요소 |
|------|-----------|
| 그룹 탐색/검색 | 카테고리 필터(주제/지역), 그룹 카드(이름, 설명, 멤버 수) |
| 그룹 상세 | 그룹 정보 헤더, 멤버 목록, 그룹 전용 피드 |
| 그룹 생성 폼 | 그룹명, 설명, 카테고리 선택, 공개/비공개 설정, 이미지 업로드 |
