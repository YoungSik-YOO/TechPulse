# PRD: 알림

## 기능 개요
사용자의 게시물에 대한 반응, 팔로우, DM 등 주요 이벤트를 실시간으로 알려주는 알림 기능입니다.

## 우선순위: 🟡 중요

## 상세 기능 명세

### F9-1. 인앱 알림
| 항목 | 내용 |
|------|------|
| 기능명 | 인앱 알림 |
| 설명 | 사용자의 활동과 관련된 이벤트(댓글, 좋아요, 리포스트, 팔로우, DM, 그룹 소식)를 앱 내 알림 센터에서 확인할 수 있다. |
| 입력 | 이벤트 발생 시 자동 생성 |
| 처리 | Firestore 실시간 리스너를 통해 새 알림 감지, 알림 문서 생성 |
| 출력 | 알림 벨 아이콘에 뱃지(읽지 않은 알림 수), 알림 목록 페이지 |
| 예외 | 알림 과다 시 → 유사 알림 그룹화 (예: '외 3명이 좋아요를 눌렀습니다') |

### F9-2. 알림 유형
| 유형 | 트리거 | 메시지 예시 |
|------|--------|-------------|
| 댓글 | 내 게시물에 댓글 작성 | "홍길동님이 회원님의 게시물에 댓글을 남겼습니다." |
| 좋아요 | 내 게시물에 좋아요 | "홍길동님이 회원님의 게시물을 좋아합니다." |
| 리포스트 | 내 게시물이 리포스트됨 | "홍길동님이 회원님의 게시물을 리포스트했습니다." |
| 팔로우 | 새로운 팔로워 | "홍길동님이 회원님을 팔로우하기 시작했습니다." |
| DM | 새 메시지 수신 | "홍길동님에게서 새 메시지가 도착했습니다." |
| 그룹 | 가입된 그룹에 새 게시물 | "AI연구 그룹에 새 게시물이 올라왔습니다." |

### F9-3. 알림 설정
| 항목 | 내용 |
|------|------|
| 기능명 | 알림 설정 |
| 설명 | 사용자는 알림 유형별로 수신 여부를 설정할 수 있다. |
| 입력 | 알림 설정 화면에서 유형별 토글 |
| 처리 | Firestore 사용자 설정 문서 업데이트 |
| 출력 | 설정에 따라 해당 유형의 알림 수신/차단 |
| 예외 | 기본값은 모든 알림 수신 |

### F9-4. 푸시 알림 (향후 확장)
| 항목 | 내용 |
|------|------|
| 기능명 | 웹 푸시 알림 |
| 설명 | 브라우저를 닫은 상태에서도 중요한 알림을 받을 수 있다. |
| 입력 | 브라우저 푸시 권한 허용 |
| 처리 | Firebase Cloud Messaging (FCM)을 통한 푸시 알림 발송 |
| 출력 | 브라우저 또는 모바일 기기에 푸시 알림 표시 |
| 예외 | 사용자가 푸시 권한 거부 시 → 인앱 알림으로 대체 |

## 화면 구성

| 화면 | 주요 요소 |
|------|-----------|
| 알림 아이콘 | 상단 내비게이션 바에 벨 아이콘 + 읽지 않은 알림 수 뱃지 |
| 알림 목록 | 알림 카드 (발신자 프로필, 알림 내용, 시간), 읽음/읽지 않은 상태 구분 |
| 알림 설정 | 알림 유형별 토글 스위치 |
